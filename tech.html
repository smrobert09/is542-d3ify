D3-ify

Divine Dom Mo-Dification

Alright, so the D3 in not really of divine origin, but looking at the neat things that have been done with it, it very well could be. Javascript libraries are many and varied. In essence, they serve a single purpose, to simplify interaction with a web page through bundling simple commands into complex functions that are much easier to call.

So I am saying that there many libraries that do what D3 does? So why should I care? Document (or DOM) maniupation is the heart of D3. However, it is so much more. Some libraries, especially those involving data manipulation and presentation, include custom syntax and markup that is required to use. D3 is different. It is standards driven. It utilizes only HTML, CSS, and SVG, three of the most basic, universal standards in technology. This means that it is all but guaranteed to function properly on any modern browser.

What is SVG you ask? Don't worry, I had the same question. SVG stands for scalable vector graphics. These I had heard about. Basically, it breaks a picture down into a set of functions (or vectors) that can be used to redraw the image. The amazing thing is, if I want to make the picture twice as big, I just tell these functions to draw twice as long as normal. The same goes if I want to shrink an image. What is even better (and makes SVGs useful inside oo a website) is that the instructions are kept in a strict XML format.

You should be saying to yourself, "Hey, so is HTML!" Thats right. Because of the XML formating, any browser already knows how to interpret the tags. It is just a matter of how these tags should be rendered visually. Thanks to SVG (supplying shape) and CSS (providing style), we are set to make all sorts of cool visualiztions and access them from almost anywhere. Neat huh.

So, do you want to see how it works? Great. Lets get started.

Below is the HTML for a basic, blank page. Nothing special here right? Just wait.

<!DOCTYPE html>
<html >
<head>
<meta charset="UTF-8"/>
<title>Test Page</title>
</head>
<body>
</body>
</html>

Now, we add the library with a link tag, and call a function to append a new tag to the body, an SVG tag.

<!DOCTYPE html>
<html >
<head>
<meta charset="UTF-8"/>
<title>Test Page</title>
</script></head>
<body>
<script src="http://d3js.org/d3.v3.min.js" charset="utf-8">

</body>
</html>

Now that we have our SVG tag, we can do all sorts of cool things. Such as draw a circle, resize the circle, or move the circle across the page.

With this data, I can run calculations to and draw circles of relative sizes as a percent of the total. I can also move those circles into groups based upon other parameters, such as type, change over previous years, or by functional department.
